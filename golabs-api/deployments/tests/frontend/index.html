GoLabs API — Documentación

              
Referencia completa de la API REST. Todos los endpoints requieren autenticación Bearer JWT salvo
                indicación contraria. Las respuestas siempre usan Content-Type: application/json.

              

                v1.0
                JSON
                JWT Bearer
                REST
              

            


            

              
Descripción general

              

                

                  

                  
Base URL

                  
/api/v1 — Todos los endpoints se montan bajo este prefijo.

                

                

                  

                  
Autenticación

                  
JWT de corta duración + refresh token. Header:
                    Authorization: Bearer <token>
                  

                

                

                  

                  
Formato

                  
Peticiones y respuestas usan Content-Type: application/json.

                

                

                  

                  
Errores

                  
Siempre incluyen un campo error con mensaje descriptivo y un
                    code opcional.
                  

                

              

            


            

              
Flujo de autenticación

              

                

                  
Cliente

                  
Usuario

                

                
→

                

                  
POST /auth/login

                  
Credenciales

                

                
→

                

                  
access_token

                  
~15 min

                

                
+

                

                  
refresh_token

                  
~7 días

                

                
→

                

                  
POST /auth/refresh

                  
Renovar

                

                
→

                

                  
Nuevo access_token

                  
Rotación automática

                

              

              
La consola renueva el token
                automáticamente ante un 401 y reintenta la petición de forma transparente.

            


            

              
Health

              

                
GET/healthz/liveLiveness probe

                

                  
Verifica que el proceso está en ejecución. No requiere autenticación.

                  

                    
Respuestas

                    

                      
200Servidor activo —
                          {"status":"ok"}

                    

                  

                

              

              

                
GET/healthz/readyReadiness probe

                

                  
Verifica la conexión a la base de datos y dependencias. No requiere autenticación.
                  

                  

                    
Respuestas

                    

                      
200Sistema completamente
                          operativo

                      
503Una o más dependencias
                          no disponibles

                    

                  

                

              

            


            

              
Autenticación

              

                
POST/auth/registerRegistro

                

                  
Crea una nueva cuenta. No requiere autenticación.

                  

                    
Body (JSON)

                    

                      
                        

                          Campo
                          Tipo
                          Req.
                          Descripción
                        

                      
                      
                        

                          username
                          string
                          Sí
                          Mínimo 3 caracteres, único
                        

                        

                          email
                          string
                          Sí
                          Correo válido y único
                        

                        

                          password
                          string
                          Sí
                          Mínimo 6 caracteres
                        

                      
                    

                  

                  

                    
Respuestas

                    

                      
201Usuario creado
                      

                      
409Usuario o correo ya
                          existe

                      
422Campos inválidos
                      

                    

                  

                

              

              

                
POST/auth/loginInicio de sesión

                

                  
Autentica al usuario y devuelve un par de tokens JWT.

                  

                    
Body (JSON)

                    

                      
                        

                          Campo
                          Tipo
                          Req.
                          Descripción
                        

                      
                      
                        

                          identifier
                          string
                          Sí
                          Correo o nombre de usuario
                        

                        

                          password
                          string
                          Sí
                          Contraseña del usuario
                        

                      
                    

                  

                  

                    
Respuesta 200

                    
{
                      "access_token": "eyJhbGciOiJIUzI1NiIs...",
                      "refresh_token": "dGhpcyBpcyBhIHJlZnJlc2...",
                      "expires_in": 900
                      }
                    

                  

                  

                    
Respuestas

                    

                      
200Autenticación exitosa —
                          access_token y refresh_token

                      
401Credenciales
                          incorrectas

                      
403Cuenta suspendida o
                          baneada

                    

                  

                

              

              

                
POST/auth/refreshRenovar token

                

                  
Emite un nuevo access token usando el refresh token. El anterior se invalida
                    (rotación de tokens).

                  

                    
Respuestas

                    

                      
200Nuevo par de tokens
                          emitidos

                      
401Refresh token inválido,
                          expirado o ya utilizado

                    

                  

                

              

            


            

              
Events

              
Ciclo de vida:
                draft → open → running → finished. Solo
                administradores pueden crear eventos y avanzar su estado.
              

              

                
GET/events/Listar eventos🔒

                

                  
Devuelve una lista paginada de todos los eventos.

                  

                    
Query params

                    

                      
                        

                          Parámetro
                          Tipo
                          Req.
                          Descripción
                        

                      
                      
                        

                          page
                          integer
                          No
                          Número de página, por defecto 1
                        

                        

                          size
                          integer
                          No
                          Registros por página, por defecto 20, máximo 100
                        

                      
                    

                  

                  

                    
Respuestas

                    

                      
200Lista paginada de
                          eventos

                      
401No autenticado
                      

                    

                  

                

              

              

                
POST/events/Crear evento🔒 admin

                

                  
Crea un nuevo evento CTF en estado draft. Requiere rol admin.

                  

                    
Body (JSON)

                    

                      
                        

                          Campo
                          Tipo
                          Req.
                          Descripción
                        

                      
                      
                        

                          name
                          string
                          Sí
                          Nombre del evento
                        

                        

                          description
                          string
                          No
                          Descripción detallada
                        

                        

                          max_team_size
                          integer
                          Sí
                          Número máximo de integrantes por equipo
                        

                        

                          starts_at
                          ISO 8601
                          Sí
                          Fecha y hora de inicio en UTC
                        

                        

                          ends_at
                          ISO 8601
                          Sí
                          Fecha y hora de cierre en UTC
                        

                      
                    

                  

                  

                    
Respuestas

                    

                      
201Evento creado —
                          devuelve el objeto con id

                      
403Requiere rol
                          admin

                      
422Campos inválidos
                      

                    

                  

                

              

            


            

              
Challenges

              

                
POST/events/{eid}/challenges/{cid}/submitEnviar
                    flag🔒

                

                  
Envía una flag para el reto especificado. Un reto solo puede resolverse una vez
                    por equipo. Si es correcta, otorga los puntos al equipo.

                  

                    
Path params

                    

                      
                        

                          Parámetro
                          Tipo
                          Descripción
                        

                      
                      
                        

                          eid
                          uuid
                          ID del evento
                        

                        

                          cid
                          uuid
                          ID del challenge
                        

                      
                    

                  

                  

                    
Respuesta 200

                    
{
                      "correct": true,
                      "points": 200,
                      "message": "Flag correcta. +200 puntos"
                      }
                    

                  

                  

                    
Respuestas

                    

                      
200Flag procesada — ver
                          campo correct

                      
409El equipo ya resolvió
                          este challenge

                      
403El evento no está en
                          estado running

                      
404Evento o challenge no
                          encontrado

                    

                  

                

              

            


            

              
Teams

              

                
POST/events/{eid}/teams/joinUnirse a equipo🔒

                

                  
Permite al usuario autenticado unirse a un equipo usando la clave de ingreso
                    generada por el líder. El evento debe estar en estado open.

                  

                    
Body (JSON)

                    

                      
                        

                          Campo
                          Tipo
                          Req.
                          Descripción
                        

                      
                      
                        

                          join_secret
                          string
                          Sí
                          Clave de ingreso generada por el equipo
                        

                      
                    

                  

                  

                    
Respuestas

                    

                      
200Usuario añadido al
                          equipo exitosamente

                      
403Evento no está en
                          estado open o equipo lleno

                      
404Clave de ingreso
                          inválida
